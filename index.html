1. SMILKit

    Videojuicer's Open Source SMIL 3.0 Engine

    http://github.com/videojuicer/smilkit-as3

  2. What is SMILKit?

    • SMILKit is a parsing and rendering engine for SMIL 3.0.
    • SMILKit is a general architecture which is implemented in language specific projects.
    • SMILKit's first implementation will be SMILKit-as3, a pure Actionscript 3.0 implementation.

  3. SMILKit Features

    • Rich Media Experiences
    • Built-in support for videos, images, audio, texts and animations
    • Extendable media handlers
    • SMIL 3.0 Support
    • Extensive SMIL layout system
    • Timing and synchronisation support with dynamic values through user states
    • Manipulatable SMIL DOM based on W3C DOM Level 2 and the Boston DOM Draft

  4. SMILKit-as3 Features

    • Video Quality of Service
    • Dynamic RTMP streaming and bit-rate switching
    • RTMP fallbacks to HTTP
    • Fast and safe buffering logic
    • Lightweight Actionscript 3.0 library with a tiny memory footprint and no dependencies
    • Retains a synchronisation rate of 200ms (4 per second) whilst keeping low CPU usage

  5. SMILKit-as3 Video QoS

    • Multiple delivery options built-in
    • RTMP (Streaming)
    • HTTP (Progressive or Streaming via Flash 10.1 only)
    • Dynamic switching between delivery options (finds the best bit-rate of the users connection)
    • Automatic RTMP fallback to HTTP (incase of bandwidth error)
    • Fast and safe buffering logic (requiring a minimum amount before playback begins)
    • Multiple bit-rates and fallbacks configured via standard W3C SMIL 3.0

  6. How SMILKit Loads a Document

    • Loads a SMIL document from the remote url
    • Parses the SMIL document via an XML parser to the SMIL DOM
    • Generates a timing graph based on the resolved assets from the DOM
    • Begins loading the assets and resolves any self-resolving assets
    • Starts the heartbeat ticking every 200ms
    • Creates a render tree from the timing graph if the assets exist at the current offset
    • Draws the assets from the render tree to the canvas via the asset handler

  8. Videojuicer Player v4 and SMILKit

    • SMILKit provides a rendered stage from a SMIL document
    • Uses the SMILKit DOM to allow manipulation to the presentation during runtime
    • First implementing media player to use the SMILKit engine
    • Player v4 provides the user interface and relays the playback commands to SMILKit
    • Addon architecture in v4 will allow SMIL DOM events to pass through to an addon
    • Player v4 adds VAST support to SMILKit through a custom asset handler
